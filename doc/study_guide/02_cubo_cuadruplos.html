2.1 cubo.py

Propósito: dado un operador y dos tipos, ¿es válida la operación y de qué tipo es el resultado?

Representado como: dict[(op, t_izq, t_der) -> t_resultado]. Si la tupla no existe en el dict, la operación es inválida.

Tipos definidos

ENTERO   = 'entero'
FLOTANTE = 'flotante'
BOOL     = 'bool'

El cubo completo (40 entradas)

OperadorTipos de operandosTipo resultadoNotas
+ - * /cualquier combo de entero/flotanteflotante si alguno es flotante; entero si ambos enterosEnsanchamiento int->float automático
< > == !=cualquier combo de entero/flotanteboolCompara escalares
=(dest, src): entero=entero, flotante=flotante, flotante=enterotipo del destinoNO permite entero=flotante (estrechamiento prohibido)

El cubo incluye la asignación. La misma función tipo_resultado('=', dest, src) sirve para:

  • r = 3 + 5 — valida asignación a variable
  • retorna E — valida que tipo(E) sea asignable al tipo_retorno
  • argumento de llamada — valida que tipo(arg) sea asignable al tipo del param

API pública

def tipo_resultado(op, tipo_izq, tipo_der):
    """Retorna el tipo de resultado, o None si la combinacion no existe."""
    return _CUBO.get((op, tipo_izq, tipo_der))

_CUBO es privado (prefijo _); sólo se accede vía tipo_resultado().

2.2 La clase Cuadruplo

@dataclass
class Cuadruplo:
    op:        str
    opIzq:     int = None
    opDer:     int = None
    resultado: int = None

Tupla de 4 campos. Unidad atómica del código intermedio.

EJEMPLOS: (+, 5000, 13000, 9000) # 5000 + 13000 -> 9000 (temp) (=, 9000, _, 1000) # 1000 = leer(9000) (GOTOF, 11000, _, 7) # si !leer(11000): IP = 7 (ERA, _, _, 'sumar') # reserva frame para sumar (GOSUB, 'sumar', _, 3) # invoca sumar, cuerpo en IP 3 (ENDFUNC, _, _, _) # pop frame, IP = ip_retorno

Los campos son direcciones virtuales (no nombres). Cada uno apunta a un slot de memoria según su rango (segmento × tipo).

2.3 TablaConstantes — dedup de literales

Cada literal del programa (e.g. 5, 3.14, "hola") recibe una dirección virtual única. Si el mismo literal aparece N veces, se reusa la misma dirección (deduplicación).

Rangos virtuales

_BASE = {
    "entero":   13000,
    "flotante": 14000,
    "letrero":  15000,
}

Atributo de clase. La VM lo importa directamente: from cuadruplos import TablaConstantes; TablaConstantes._BASE. Única fuente de verdad entre compilador y VM.

Métodos principales

def direccion_de_numerica(self, lexema, tipo):
    """Devuelve direccion (existente o nueva) para entero o flotante."""

def direccion_de_letrero(self, lexema):
    """Devuelve direccion (existente o nueva) para string."""

Ejemplo de dedup

i = 1;       # '1' -> nueva direccion 13000
j = 1;       # '1' dedup -> misma 13000
i = i + 1;   # otro '1' dedup -> misma 13000
i = -1;      # '-1' es lexema DISTINTO -> nueva direccion 13001

La dedup es por lexema (string), no por valor numérico. "1" y "1.0" son distintos (uno entero, uno flotante). "5" y "-5" también.

2.4 GeneradorCuadruplos

Clase principal. Maneja:

  • 5 pilas auxiliares (todas list de Python)
  • 1 fila de cuádruplos (la salida)
  • Métodos para emitir cuádruplos
  • Método de backpatch para resolver saltos pendientes

Estado

def __init__(self, mem, cte):
    self.mem = mem              # AsignadorMemoria (referencia)
    self.cte = cte              # TablaConstantes (referencia)
    self.pila_operandos  = []   # direcciones pendientes
    self.pila_tipos      = []   # tipos paralelos
    self.pila_operadores = []   # operadores pendientes (+, -, *, etc.)
    self.pila_saltos     = []   # indices de cuadruplos pendientes de backpatch
    self.fila            = []   # lista de Cuadruplos (la salida)
    self._ultimo_propagado = False
    self.idx_goto_main   = None # indice del GOTO inicial

Por qué pilas paralelas (pila_operandos y pila_tipos): cuando emites un cuádruplo binario necesitas dos cosas: la dirección del operando (para meter en el cuádruplo) y el tipo (para consultar el cubo). Las pilas se mueven en sincronía — push juntas, pop juntas.

El método clave: pop_y_emitir_binario

def pop_y_emitir_binario(self):
    """Pop del operador + 2 operandos+tipos. Consulta el cubo.
    Asigna temporal, emite cuadruplo, push del temporal."""
    op = self.pila_operadores.pop()
    d_der = self.pila_operandos.pop()
    t_der = self.pila_tipos.pop()
    d_izq = self.pila_operandos.pop()
    t_izq = self.pila_tipos.pop()

    # Propagacion silenciosa de errores
    if t_izq == "error" or t_der == "error":
        self.pila_operandos.append(None)
        self.pila_tipos.append("error")
        return "error"

    # Consulta el cubo
    t_res = tipo_resultado(op, t_izq, t_der)
    if t_res is None:
        return "error"

    # Asigna temporal y emite
    d_res = self.mem.nuevo_temporal(t_res)
    self.fila.append(Cuadruplo(op=op, opIzq=d_izq, opDer=d_der, resultado=d_res))
    self.pila_operandos.append(d_res)
    self.pila_tipos.append(t_res)
    return t_res

Backpatch — resolver forward references

def backpatch(self, indice, target):
    """Muta el campo resultado del cuadruplo en self.fila[indice]."""
    self.fila[indice].resultado = target

Resuelve forward references: emites (GOTOF, dir, _, None) sin saber a dónde saltar, guardas el índice. Después cuando ya sabes, llamas backpatch(indice, target) para rellenar.

2.5 Tips de examen

"¿Por qué se llaman cuádruplos y no triples?"

Porque tienen 4 campos: operador + 2 operandos + resultado. Es código intermedio de 3 direcciones (3AC): "3" se refiere a las direcciones, no a los campos.

"¿Por qué TablaConstantes._BASE es atributo de clase?"

Porque la VM lo importa directamente. Si fuera de instancia, tendría que crear una TablaConstantes() sólo para leer los rangos. Como atributo de clase es única fuente de verdad.

"¿Qué hace pop_y_emitir_binario si hay error de tipos?"

Empuja sentinel (None, "error") para mantener balance. Si NO empujara, las pilas quedarían desbalanceadas y la siguiente operación popearía operandos equivocados. El error se reporta una vez y se propaga silenciosamente (_ultimo_propagado=True).

Las 5 pilas: las 3 de expresión (operandos, tipos, operadores) viven mientras dura UNA expresión. Las 2 de forward refs (saltos y llamadas) viven mientras dura UNA construcción / UNA llamada. Las 5 quedan vacías al cerrar su contexto — esa es la invariante.
