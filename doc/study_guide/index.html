Study Guide — Compilador Patito (Python)

Esta guía cubre los 6 archivos fuente del compilador Patito en Python. Cada archivo se explica con su propósito, qué provee, sus dependencias internas, walkthrough del código, y cómo se conecta con los demás.

Pensada para preparación de defensa de proyecto: cada sección cierra con tips de examen.

Tabla de contenidos

  • 1. Gramática Patito.g4 — léxico + sintaxis + ANTLR
  • 2. cubo.py + cuadruplos.py — tipos + código intermedio
  • 3. semantico.py — clases + AsignadorMemoria + los 24 PNs
  • 4. vm.py — Frame + MaquinaVirtual + los 12 opcodes
  • 5. patito.py + Flujo end-to-end — driver + trace real

Arquitectura global

El compilador Patito procesa un .patito en 5 etapas a través de un pipeline lineal sin ciclos:

+----------------------------------------------------------------------+ | patito.py (driver) | | parse args -> instancia lexer/parser -> llama analyzer -> opt VM | +----------------------------------------------------------------------+ | v +-----------+ +-----------+ +-----------------+ +-----------+ | ANTLR | | generated/| | semantico.py | | vm.py | | Patito.g4 |-->| PatitoLex |-->| SemanticAnaly- |-->| Maquina | | (grammar) | | PatitoPars| | zer + 24 PNs | | Virtual | +-----------+ +-----------+ +-----------------+ +-----------+ | +-----------+-----------+ v v v +-------+ +----------+ +--------------+ | cubo | | cuadrup- | | (gen. fila + | | sem. | | los.py | | cte para VM) | +-------+ +----------+ +--------------+

Cadena de imports (sin ciclos)

patito.py | importa +-> semantico.SemanticAnalyzer +-> vm.MaquinaVirtual +-> generated.PatitoLexer / PatitoParser semantico.py | importa +-> cuadruplos.Cuadruplo / TablaConstantes / GeneradorCuadruplos +-> cubo.tipo_resultado +-> generated.PatitoListener / PatitoParser (isinstance del padre) cuadruplos.py | importa +-> cubo.tipo_resultado vm.py | importa +-> semantico.AsignadorMemoria (para ._BASE) +-> cuadruplos.TablaConstantes (para ._BASE) cubo.py +-> (no importa nada; expone tipo_resultado + constantes)

Cómo usar esta guía

  1. Lee esta página primero (arquitectura global)
  2. 01 Grammar — sin entender qué reconoce el parser, no entiendes qué dispara cada PN
  3. 02 Cubo + Cuadruplos — primitivas usadas por el analizador semántico. Cortas pero importantes.
  4. 03 Semántico — el más grande (766 líneas). 24 PNs organizados en 5 grupos.
  5. 04 VM — cómo se ejecutan los cuádruplos. 12 opcodes + 3 reglas de memoria.
  6. 05 Driver + Flujo — patito.py y trace end-to-end de un programa real.

Si vas a hacer el quiz interactivo después, lee primero todo el guide y MARCA mentalmente las cosas que no te queden claras. Esas son las que el quiz va a reforzar.
