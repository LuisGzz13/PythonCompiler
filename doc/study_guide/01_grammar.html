1.1 Estructura general

Un programa Patito sigue siempre esta forma:

programa NOMBRE ;
vars? funcs* inicio cuerpo fin
programa --- id --- ; --- vars? --- funcs* --- inicio --- cuerpo --- fin | | | | v v v v "programa" vars lista de main block globales funciones

1.2 Reglas estructurales

Programa

programa
    : PROGRAMA ID PCOMA varsOpc funcsOpc INICIO cuerpo FIN
    ;

Declaraciones de variables

vars            : VARS declaracionList ;
declaracionList : declaracion declaracionList | declaracion ;
declaracion     : idLista DPUNTOS tipo PCOMA ;
idLista         : ID idResto ;
idResto         : COMA ID idResto | epsilon ;
tipo            : ENTERO | FLOTANTE ;

Una declaración puede listar múltiples IDs: vars i, j, k : entero;.

Funciones

funcs
    : tipoFunc ID PIZQ paramsOpc PDER LLAVEIZQ varsOpc cuerpo LLAVEDER PCOMA
    ;

tipoFunc   : NULA | tipo ;
paramsOpc  : paramLista | epsilon ;
paramLista : ID DPUNTOS tipo paramResto ;
paramResto : COMA ID DPUNTOS tipo paramResto | epsilon ;

tipoFunc admite NULA o un tipo (entero/flotante). Las funciones pueden ser nula, entero o flotante — nunca bool (bool no está en tipo).

Llamadas a funciones

llamada  : ID PIZQ argsOpc PDER ;
argsOpc  : argLista | epsilon ;
argLista : expresion argResto ;
argResto : COMA expresion argResto | epsilon ;

1.3 Expresiones con precedencia

La gramática usa 3 niveles para implementar precedencia:

expresion : exp relOpc ;
relOpc    : opRel exp | epsilon ;
opRel     : MAYOR | MENOR | DISTINTO | IGUAL2 ;

exp     : exp MAS termino | exp MENOS termino | termino ;
termino : termino POR factor | termino ENTRE factor | factor ;
factor  : PIZQ expresion PDER
        | llamada
        | signoOpc ID
        | signoOpc cte ;

signoOpc : MAS | MENOS | epsilon ;
cte      : CTE_ENT | CTE_FLOT ;

"¿Cómo implementa Patito la precedencia de operadores?"

Respuesta: con jerarquía gramatical. termino se evalúa antes que exp, y factor antes que termino. Por eso 2 + 3 * 4 se reconoce como 2 + (3 * 4): la regla termino -> termino POR factor atrapa la multiplicación antes de que exp vea el +.

Los relacionales están en el nivel más alto (menor precedencia) — bien para condiciones.

Recursión por la izquierda

ANTLR4 maneja recursión izquierda automáticamente (a diferencia de yacc). Esto da asociatividad izquierda: a - b - c se evalúa como (a - b) - c.

1.4 Statements

cuerpo        : LLAVEIZQ estatutoList LLAVEDER ;
estatutoList  : estatuto estatutoList | epsilon ;

estatuto
    : asigna
    | condicion
    | ciclo
    | llamada PCOMA
    | imprime
    | retorno    /* agregado en la implementacion de retorna */
    ;

Asignación

asigna : ID IGUAL expresion PCOMA ;

Condición (si/sino)

condicion : SI PIZQ expresion PDER cuerpo sinoOpc PCOMA ;
sinoOpc   : SINO cuerpo | epsilon ;

Ciclo (mientras)

ciclo : MIENTRAS PIZQ expresion PDER HAZ cuerpo PCOMA ;

Escribe

imprime  : ESCRIBE PIZQ impLista PDER PCOMA ;
impLista : impElem impResto ;
impResto : COMA impElem impResto | epsilon ;
impElem  : expresion | LETRERO ;

Retorno

retorno : RETORNA expresion PCOMA ;

1.5 Tokens

Palabras reservadas (13)

PROGRAMA : 'programa' ;     SI       : 'si' ;
INICIO   : 'inicio' ;       SINO     : 'sino' ;
FIN      : 'fin' ;          MIENTRAS : 'mientras' ;
VARS     : 'vars' ;         HAZ      : 'haz' ;
ENTERO   : 'entero' ;       NULA     : 'nula' ;
FLOTANTE : 'flotante' ;     RETORNA  : 'retorna' ;
ESCRIBE  : 'escribe' ;

Operadores (9)

MAS  : '+'    MAYOR    : '>'
MENOS: '-'    MENOR    : '<'
POR  : '*'    DISTINTO : '!='
ENTRE: '/'    IGUAL2   : '=='
              IGUAL    : '='

Patito tiene 4 operadores relacionales (no 6 como C): <, >, ==, !=. No hay <= ni >=.

Puntuación (7)

PCOMA : ';'   COMA : ','   DPUNTOS : ':'
PIZQ  : '('   PDER : ')'   LLAVEIZQ : '{'  LLAVEDER : '}'

Literales

ID       : [a-zA-Z] [a-zA-Z0-9_]* ;
CTE_FLOT : [0-9]+ '.' [0-9]+ ;   /* ANTES DE CTE_ENT */
CTE_ENT  : [0-9]+ ;
LETRERO  : '"' ~["\r\n]* '"' ;
WS       : [ \t\r\n]+ -> skip ;

¿Por qué CTE_FLOT antes de CTE_ENT? El lexer prioriza el token declarado primero ante ambigüedad. Si CTE_ENT fuera primero, 3.14 se parsearía como CTE_ENT('3') '.' CTE_ENT('14') — error de sintaxis.

1.6 Cómo ANTLR genera el parser

Comando (documentado en README.md):

( cd grammar && antlr -Dlanguage=Python3 -visitor Patito.g4 -o ../generated )

Esto produce 4 archivos en generated/:

ArchivoQué provee
PatitoLexer.pyLexer: texto a tokens
PatitoParser.pyParser: tokens a árbol sintáctico
PatitoListener.pyClase base con enterX/exitX vacíos por cada regla
PatitoVisitor.pyPatrón visitor (no usado en Patito)

SemanticAnalyzer en semantico.py hereda de PatitoListener y sobreescribe los enterX/exitX que le interesan. Los demás permanecen vacíos.

1.7 Tips de examen

"¿Por qué Patito no tiene <= ni >=?"

Decisión deliberada. Para agregarlos: token nuevo en lexer + alternativa en opRel + entradas en _CUBO + handling en VM.

"¿Qué pasa si en un programa no hay vars?"

varsOpc matchea la rama epsilon (vacía), y enterDeclaracion nunca dispara. El programa compila normal sin globales.

"¿Cuál es la gramática de una función?"

tipoFunc ID PIZQ paramsOpc PDER LLAVEIZQ varsOpc cuerpo LLAVEDER PCOMA. Notar dos pares de llaves: las externas envuelven la función completa (params + vars + cuerpo), las internas (dentro de cuerpo) envuelven los estatutos.
