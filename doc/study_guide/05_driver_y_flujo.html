5.1 ContadorErrores

class ContadorErrores(ErrorListener):
    """ErrorListener custom: imprime errores a stderr Y mantiene contador."""
    def __init__(self):
        super().__init__()
        self.errores = 0

    def syntaxError(self, recognizer, offendingSymbol, line, col, msg, e):
        print(f"linea {line}:{col}  {msg}", file=sys.stderr)
        self.errores += 1

Sin esto, ANTLR imprime errores pero el parser sigue. No sabríamos al final si hubo errores para decidir exit code. Los errores semánticos están en SemanticAnalyzer.error() (un counter aparte llamado analizador.errores).

5.2 main() — el pipeline

def main():
    # 1. Validar args
    if len(sys.argv) < 2:
        print("Uso: python patito.py [--cuadruplos] [--dir] [--ejecutar] archivo.patito",
              file=sys.stderr)
        sys.exit(2)

    args = sys.argv[1:]
    imprimir_cuadruplos = "--cuadruplos" in args
    ejecutar = "--ejecutar" in args
    imprimir_directorio = "--dir" in args

    archivos = [a for a in args if not a.startswith("--")]
    if len(archivos) != 1:
        print("Uso: ...", file=sys.stderr)
        sys.exit(2)

    # 2. Pipeline de lectura y analisis
    archivo = archivos[0]
    stream = FileStream(archivo)
    lexer = PatitoLexer(stream)
    tokens = CommonTokenStream(lexer)
    parser = PatitoParser(tokens)

    # 3. Custom error listener
    err_sintaxis = ContadorErrores()
    lexer.removeErrorListeners()
    lexer.addErrorListener(err_sintaxis)
    parser.removeErrorListeners()
    parser.addErrorListener(err_sintaxis)

    # 4. Parse
    tree = parser.programa()
    if err_sintaxis.errores > 0:
        sys.exit(1)

    # 5. Analisis semantico
    analizador = SemanticAnalyzer()
    walker = ParseTreeWalker()
    walker.walk(analizador, tree)
    if analizador.errores > 0:
        sys.exit(1)

    # 6. Flags opcionales
    if imprimir_cuadruplos:
        analizador.cte.imprimir()
        print()
        analizador.gen.imprimir_fila()

    if ejecutar:
        try:
            MaquinaVirtual(analizador).ejecutar()
        except RuntimeError as e:
            print(f"VM error: {e}", file=sys.stderr)
            sys.exit(1)

    if imprimir_directorio:
        analizador.imprimir_directorio()

    sys.exit(0)

5.5 Flujo end-to-end: trace de un programa real

Usamos un programa con un mientras para que el trace toque todas las etapas:

programa demo;
vars i, s : entero;
inicio
{
    i = 1;
    s = 0;
    mientras (i < 4) haz {
        s = s + i;
        i = i + 1;
    } ;
    escribe(s);
}
fin

5.6 Tips de examen

"¿Por qué patito.py instala su propio ErrorListener?"

Porque el ErrorListener default de ANTLR imprime a stderr pero NO trackea. Sin el custom, no podríamos decidir exit code 0 o 1 al final. Además el custom imprime el formato "linea L:C msg" que queremos.

"¿Cuándo retorna exit code 1 vs exit code 2?"

Exit 2: usuario mal-invocó el comando (faltan args). Exit 1: error de compilación (sintáctico, semántico) o de ejecución (RuntimeError de la VM). Exit 0: todo OK.

"Cuál es el orden EXACTO de procesamiento desde un archivo .patito hasta stdout?"

1) FileStream lee el archivo
2) PatitoLexer convierte a tokens
3) CommonTokenStream buffer
4) PatitoParser construye el árbol (raiz: regla programa)
5) Si hay errores sintácticos -> exit 1
6) ParseTreeWalker recorre el árbol invocando los PNs del SemanticAnalyzer
7) Si hay errores semánticos -> exit 1
8) Si --cuadruplos: imprime cte + fila
9) Si --ejecutar: instancia MaquinaVirtual y llama ejecutar()
10) Si --dir: imprime func_dir
11) exit 0

Para revisar el trace de un programa: usa --cuadruplos --ejecutar juntos. Imprime los cuádruplos PRIMERO (para que veas lo que se va a ejecutar), luego ejecuta y ves el output. Ideal para debugging.
